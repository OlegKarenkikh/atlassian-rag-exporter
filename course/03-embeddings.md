# Глава 3: Как работают эмбеддинги

![Текст превращается в вектор](img/03-embeddings.svg)

## Компьютер не понимает слова

Нейросети работают с числами. Чтобы искать по смыслу, нужно превратить текст в числа. Это делают **модели эмбеддингов**.

## Что такое вектор?

Представьте карту: каждый документ — точка на карте. Похожие документы стоят рядом. Документы про «SSO» — в одном районе, про «деплой» — в другом.

Вектор — координаты точки, но не в 2D, а в **384-мерном пространстве**. Модель `intfloat/multilingual-e5-large` даёт вектор из 384 чисел для каждого фрагмента.

## Как ищем?

Вопрос тоже превращается в вектор. Затем находим **ближайшие** векторы документов по **косинусному расстоянию** (угол между векторами: 0 = одинаковые, 1 = противоположные).

## Почему multilingual-e5?

Модель хорошо работает с **русским языком** и смешанными текстами (RU + EN термины). E5 требует префикс:

```python
# При индексировании документов:
"passage: Как настроить SSO через SAML?"

# При поиске (запрос пользователя):
"query: SSO SAML настройка"
```

Добавляется автоматически нашим `SentenceTransformersEmbedder`.

## Нарезка на фрагменты (Chunking)

Целую страницу нельзя превратить в один вектор — слишком длинная. `chunk_document()` делит текст на фрагменты по **500 токенов** с перекрытием **50 токенов**:

```
[Фрагмент 1: токены 0-500]
                  [Фрагмент 2: токены 450-950]
                                [Фрагмент 3: токены 900-1400]
```

Перекрытие гарантирует, что ответ на стыке двух фрагментов всё равно будет найден.
