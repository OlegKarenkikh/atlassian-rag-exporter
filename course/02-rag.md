# Глава 2: Что такое RAG

![Как работает RAG](img/02-rag.svg)

## Проблема: LLM не знает ваши документы

Большие языковые модели (GPT-4, LLaMA, Mistral) обучены на публичных данных из интернета. Они не знают, что написано в вашей внутренней Confluence. Если спросить LLM «Как у нас настраивается SSO?», она либо скажет «не знаю», либо **придумает ответ** — это называется галлюцинация.

## Решение: RAG

**RAG = Retrieval-Augmented Generation** — генерация с поиском.

Идея проста: прежде чем спросить LLM, сначала **найдите** релевантные документы из вашей базы и **вложите** их в запрос. Тогда модель отвечает по реальным документам, а не из головы.

## Пиплайн за 60 секунд

1. Пользователь задаёт вопрос: «Как настроить SSO через SAML?»
2. **Эмбеддер** превращает вопрос в вектор (384 числа)
3. **Векторная БД** ищет 5 ближайших документов
4. **LLM** получает: вопрос + найденные фрагменты
5. **LLM** отвечает, опираясь только на реальные документы

## Метрики качества

| Метрика | Что измеряет | Идеал |
|---------|-------------|-------|
| **Faithfulness** | Ответ основан на контексте? | 1.0 |
| **Context Relevance** | Найденные фрагменты по теме? | 1.0 |

`RAGTester` вычисляет их автоматически после каждого вопроса.

## Без RAG vs С RAG

```
Без RAG:  «SSO настраивается в разделе Administration...»
          (придумано, может быть неверным)

С RAG:    «Согласно странице SSO Configuration Guide:
           User Management -> SAML 2.0 -> Шаг 1: ...»
          (точно по вашей документации)
```
