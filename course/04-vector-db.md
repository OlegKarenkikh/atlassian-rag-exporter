# Глава 4: Векторные базы данных

![8 поддерживаемых векторных БД](img/04-vector-db.svg)

## Чем отличается от обычной БД?

Обычная БД ищет **точные совпадения**: `WHERE title = 'SSO'`. Она не найдёт «Настройка единого входа», если искали «SSO».

Векторная БД ищет **по смыслу**: кладёте вней векторы документов, задайте «найди 5 ближайших» — получите семантически похожие документы.

## Структура записи

Каждый документ в БД — три вещи:
- **ID** — `page_12345_chunk_0`
- **Вектор** — 384 числа (embedding)
- **Payload** — текст, заголовок, space_key, URL

## Какую БД выбрать?

| БД | Лучше для | Сложность |
|----|-----------|----------|
| **ChromaDB** | Локально, тесты | Минимальная |
| **Qdrant** | Продакшн, скорость | Средняя |
| **pgvector** | Уже есть PostgreSQL | Средняя |
| **Weaviate** | Гибридный поиск | Высокая |
| **Pinecone** | Managed cloud | Средняя |
| **OpenSearch** | Уже есть ES/OS | Высокая |
| **Milvus** | Большой объем | Высокая |
| **Redis Stack** | Уже есть Redis | Средняя |

**Рекомендация для старта:** ChromaDB — не нужен Docker, работает в папке:

```yaml
vector_store:
  backend: chromadb
  path: ./chroma_data
  collection: confluence_docs
```

## Инкрементальное обновление

Экспортёр хранит `.sync_state.json` с временем последней синхронизации — при повторном запуске тянет только изменённые страницы.
