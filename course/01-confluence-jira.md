# Глава 1: Что такое Confluence и Jira

![Atlassian: Confluence и Jira](img/01-atlassian.svg)

## Представьте корпоративную библиотеку

Когда компания растёт, знания начинают теряться — кто-то уволился, кто-то забыл, где лежит инструкция. **Confluence** — это корпоративная Википедия: место, где команда пишет статьи, инструкции, спецификации и FAQ.

**Jira** — трекер задач. Здесь живут баг-репорты, фичи, спринты и комментарии к ним. Оба продукта выпускает компания **Atlassian**.

## Почему они нам нужны?

Оба продукта хранят огромное количество знаний. Проблема — найти нужное среди тысяч страниц. Наш экспортёр скачивает все эти данные и делает их доступными для умного поиска с ИИ.

## Как получить данные?

Оба продукта имеют **REST API** — это как меню в ресторане: отправляешь HTTP-запрос, получаешь JSON. Confluence v2 использует курсорную пагинацию — быстрее до 30x по сравнению со старым API.

## Аутентификация: 8 способов

| Способ | Когда использовать |
|--------|-----------------|
| `token` | Email + API-токен (Confluence Cloud) |
| `pat` | Personal Access Token (Server/Data Center) |
| `oauth2` | OAuth 2.0 Bearer (enterprise) |
| `oidc` | OpenID Connect |
| `kerberos` | Корпоративный Kerberos/NTLM |
| `cookie` | Браузерная сессия (Server) |
| `basic` | HTTP Basic Auth |
| `oidc_cc` | OIDC Client Credentials |

**Самый простой старт:** создайте токен на https://id.atlassian.com/manage-profile/security/api-tokens

```yaml
auth:
  type: token
  email: you@company.com
  token: ATATT3xFfGF0...
```

## Что скачивается?

- **Страницы** — полный HTML с изображениями и вложениями
- **Вложения** — изображения, PDF, диаграммы (SHA-256 дедупликация)
- **Jira-тикеты** по JQL со всеми комментариями в формате ADF

> **ADF (Atlassian Document Format)** — внутренний формат Jira для текста. Наш парсер переводит его в Markdown автоматически.
